# 變數說明（codebook）

本文件說明 `Data/clean_data_<knit 日期>.csv` 各欄位的意義、單位與計算方式。
檔名中的日期由 R Markdown 在 knit 當天以 `Sys.Date()` 自動產生，不固定為某一天。

## 資料粒度

每一列代表臺北市一個行政區，共 12 列。CSV 預設按 `危建總數` 降冪排序（這是 R 報告產製順序，方便閱讀）。
12 個行政區為：中正、大同、中山、松山、大安、信義、內湖、南港、士林、北投、文山、萬華。

## 欄位

| 欄位 | 單位 | 說明 |
| --- | --- | --- |
| `district` | — | 行政區名稱。原始資料中「台北市」與「臺北市」用字、以及行政區寫法的細節差異已在清理階段統一為「臺」字版本。 |
| `地震危建數` | 筆 | 該行政區列管於臺北市地震危險建築物資料的筆數。來源：data.taipei 資料集 `a6e8f08e-ec2a-4be7-a762-54452b0c27df`。 |
| `海砂屋數` | 筆 | 該行政區列管於臺北市高氯離子混凝土建築（俗稱海砂屋）資料的筆數。來源：data.taipei 資料集 `15487e1f-a86e-4481-8ae9-3c331db5e3d4`。 |
| `都更案件數` | 件 | 該行政區涉及的不重複都市更新案件數，依案件名稱（`title`）去重後計算。來源：台北市都市發展局都市更新地圖（`map.udd.gov.taipei`），由後端 `/api/urban-update` 以 `case_count` 欄位提供。處理方式見下方「跨區案件」說明。 |
| `危建總數` | 筆 | **衍生欄位**：`地震危建數 + 海砂屋數`。 |
| `都更追趕率` | % | **衍生欄位**：`都更案件數 / 危建總數 × 100`。當 `危建總數` 為 0 時記為 `NA`，避免除以零。 |
| `都更缺口` | 件 | **衍生欄位**：`都更案件數 − 危建總數`。負值代表該區都更案件量少於危建棟數。 |

## 清理過程的處理選擇

**跨區都更案件**：原始都更資料中部分案件的行政區欄位寫法為 `士林區,北投區`，代表單一案件橫跨多個行政區。清理時以 `separate_rows` 拆開，每個涉及的行政區都計入一次案件。這意味著一筆跨兩區的案件會在兩個區各被計算一次，各區案件數加總會略高於原始案件總數。這是有意的設計，因為以行政區為比較單位時，跨區案件確實對兩區都產生影響。

**「台」/「臺」用字**：原始資料中「台北市」與「臺北市」混用，本研究以一個 `normalize_name` 函數把字串統一為「臺」字版本後再合併，避免同一行政區因用字差異被切成兩列。

**缺漏值**：行政區為空、或案件數為 NA 的紀錄會被移除（無法歸類到任何行政區，或無法計算追趕率）。地震危建與海砂屋資料的觀測值若行政區清理後仍為 NA，同樣不納入。

## 資料品質限制

**地址解析覆蓋率**：地震危建原始資料的 `building_location` 欄位是半結構化文字，混合了路／街／巷／弄、範圍門牌（`16至22號`、`86號至94號`、`118-5號`）、頓號枚舉（`2、4、6號`）、奇偶區段註記（`(雙)`、`(單)`）、括號備註、樓層描述等多種寫法。清洗腳本以正則表達式逐一展開這些寫法為單棟地址，本次清洗結果中所有紀錄均能至少解析出一個可送入地理編碼的地址。後續資料更新若出現新型寫法，仍可能需要擴充規則。

**地理編碼成功率**：解析後的地址透過 ArcGIS Geocoding API 補上經緯度。少數位於巷弄深處或門牌號無法對應到實際座標的地址會回傳查無資料，這些點位的 longitude / latitude 會被記為 0。地圖視覺化時會過濾掉這些零座標點。

**資料時效**：本檔案的數據反映執行清理當日（檔名日期）的台北市開放資料快照。原始資料更新時，重跑 R 報告會抓到最新版本，數值可能與本檔案略有差異。

## 指標的解讀限制

「都更追趕率」是相對指標，不是絕對的處理完成率。一筆都更案件可能涵蓋多棟建物，也可能仍處於計畫核定階段尚未動工；一棟危建可能對應多筆都更案件（不同期程、不同範圍劃定），也可能完全未進入都更流程。因此某行政區追趕率超過 100% 不代表危建問題已被處理完畢，應將此指標視為「都更案件量是否與危建壓力大致對齊」的概略觀察，而非處理進度的精確衡量。
